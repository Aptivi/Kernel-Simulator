| --------------- +  - >一般< -  + --------------- |

 - 如果您想参与内核模拟器，必须遵循以下条件：

1.如果要修改/重新编码核心内核，请确保正确并仔细地编写更改，确保不损坏核心内核功能。
2.如果要在任何地方添加/修改/删除/重新编码某些所需功能，请仔细编写更改。
3.像其他想要为内核模拟器做出贡献的开发人员一样，必须测试更改。我们不希望得到结果，只测试，并说它经过测试。如果您想提供结果，我们仍然接受您的拉取请求。
4.如果您尚未对其进行测试，并且您向我们发送了拉取请求，我们将测试代码，如果我们发现：

    - 您的代码正常运行，我们将接受您的拉取请求
    - 您的代码无效，我们将关闭您的拉取请求

5.如果您尚未测试，并且已准备好发送给我们，请先测试代码。
6.如果您想修改文档或编码以外的其他部分，或添加/新文档的部分内容，请随意。你可以通知我们。

 - 移植到除Assembly（ASM，NASM，...）或C / C ++语言之外的其他操作系统或编程语言：

您可能必须将内核模拟器移植到其他操作系统，如Linux，macOS等。此外，还有其他编程语言，而不是C / C ++或汇编语言。

移植到ASM / C / C ++取决于我们所有者。我们不支持在我们的官方内核ASM / C / C ++可启动版本之前向他们提供任何端口，并且有关它们的任何问题都将立即被拒绝。

 - 如果要在Visual Basic上将内核模拟器移植到Linux（已测试）或macOS（未经测试）：

1.安装Mono，MonoDevelop和libmono-microsoft-visualbasic10.0-cil软件包。你可以在他们的官方网站上找到如何。
2.下载KS源代码
3.使用MonoDevelop打开解决方案，然后运行该程序。
4.在KS回购中报告有关问题部分的任何问题。

 - 如果要将内核模拟器移植到Sun OS，BSD，Solaris等（未经测试）：

本节正在建设中。如果您有说明，请提供帮助。

注意：如果您想将内核模拟器移植到不同于Assembly或C / C ++的其他编程语言，请随意。
注意：使用Assembly和C / C ++将内核模拟器内部的内核移植到Real Computer / VirtualBox取决于我们。
注意：将模拟器内部的内核移植到Android / iOS即可。您可以进行适当的修改。
